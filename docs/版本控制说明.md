# 版本控制说明

## 当前状态

- 默认分支：`main`
- 当前基线：`cfead91 chore: establish project baseline`
- 本地 Git 仓库已初始化
- `node_modules`、`runtime-data`、构建目录和缓存不会提交

## 常用命令

```bash
git status
git log --oneline --decorate --graph -10
git diff
git add src docs
git commit -m "feat: describe the change"
```

## 提交建议

- `feat:` 新增功能
- `fix:` 修复问题
- `style:` 界面和样式调整
- `refactor:` 重构但不改变功能
- `docs:` 文档更新
- `chore:` 工程配置或依赖调整

每次提交前建议运行：

```bash
npm run check
npm run doctor
```

## 回滚方式

查看提交：

```bash
git log --oneline
```

查看某次提交的内容：

```bash
git show <commit-id>
```

恢复到某个版本前，建议先创建备份分支：

```bash
git branch backup-before-rollback
```

## 远程仓库

当前只建立了本地版本控制，尚未配置 GitHub、GitCode 或其他远程仓库。配置远程仓库后可以使用：

```bash
git remote add origin <repository-url>
git push -u origin main
```
