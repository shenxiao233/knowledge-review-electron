# WebDAV 单向备份

应用使用坚果云 WebDAV 保存本地业务数据备份。WebDAV 只作为上传目标，本地数据是唯一来源。

## 首次配置

1. 打开“设置 -> 存储”。
2. WebDAV 地址保持为 `https://dav.jianguoyun.com/dav/`。
3. 填写坚果云登录账号或邮箱，不是应用名称。
4. 填写坚果云应用密码。
5. 点击“测试连接”，再点击“保存配置”。
6. 开启“启用坚果云备份”和“每小时自动备份”。

配置保存后，地址、账号和远程文件夹会以灰色只读状态显示；需要变更时点击“编辑配置”。应用密码在界面中显示为掩码，实际值由 Electron 主进程使用系统安全存储加密保存，不会写入 `state.json`、浏览器 `localStorage`、日志或远程备份内容。

## 备份规则

- 应用启动时只加载本地数据，不会从 WebDAV 下载或覆盖本地数据。
- 新建卡片、编辑文章、复习评分、删除内容等操作只保存到本地，不会立即发起网络请求。
- 开启自动备份后，每 60 分钟上传一次完整的 `state.json` 和 `manifest.json`。
- “立即备份到坚果云”只上传当前本地快照，可随时手动执行。
- 网络不可用时应用仍可正常使用本地数据；备份失败会在下一次定时备份或手动备份时重试。

## 备份记录

- 设置页会保留最近 20 次手动或自动备份记录，包含时间、触发方式与成功/失败状态。
- 记录仅保存备份结果和必要的错误说明，不包含应用密码或卡片内容。

## 备份内容

备份包含文档、文件夹、卡片、卡组、FSRS 状态、复习事件、打卡记录、设置、学习计划和回收站内容。

不会备份 Electron 窗口状态、GPU 缓存、运行日志和 `node_modules`。当前图片功能保存的是 Markdown/HTML 中的图片地址；本地图片附件如需备份，需要后续增加附件上传机制。

## 手动恢复

当前版本不自动从 WebDAV 恢复数据，也不会用远程内容覆盖本地数据。需要恢复时，建议先从坚果云下载 `state.json`，使用应用的数据导入/恢复流程处理，并先导出本地数据作为安全备份。
